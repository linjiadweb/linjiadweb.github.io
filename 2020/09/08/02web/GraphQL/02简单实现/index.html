02简单实现

linjiad 2020-09-08 10:31:49

简答实现

用express创建一个服务端

初始化一个项目

新建一个graphql_express文件夹

npm init –y

安装依赖包

npm install –save -D express express-graphql graphql

一个简单的示例

打开http://localhost:3000/graphql

查询处输入

query{hello}

多个条件

  • 在root中添加新的数据
  • root中添加renturn的返回值

自定义类型

  • myType是我们自定义的类型
    • 用type定义类型
    • 把myType放到Query（account）
    • account定义到root中

这时候可以访问特定字段

query{
account {
name
age
}
}

查询条件

根据查询条件查询

  • getClassMates(classNo: Int!):[String]
    • 定义查询方法
    • 括号内定义参数和参数类型
  • getClassMates({classNo}){})
    • 方法的返回值

query{
getClassMates(classNo:31)
}

查询条件包含查询条件

根据查询条件查询

  • 定义查询类型Account
  • account(username:String):Account
    • 定义查询方法
    • 括号内定义参数和参数类型
    • 返回类型
  • account({username})
    • 方法的返回值

query{
account(username:”张三”){
name
sex
age
salary(city:”南京”)
}
}

客户端访问

  • 首先创建一个public空文件夹
  • 之后配置让node能返回这个静态文件
    • app.use(express.static(“public”))
  • 再public下创建index.html

把城市也变成参数

使用Mutations修改数据

  • 新建一个增加类——————input AccountInput
  • 定义新增方法——————createAccount(input:AccountInput):Account
  • 实现新增方法——————createAccount({input})
  • GraphiQL中必须含有查询类type Query，否则会报错

mutation{
createAccount(input:{
id:”2”,
name:”linjiad2”,
age:18,
sex:”男”,
department:”程序员”
}){
name
}
}
mutation{
updateAccount(id:”1”,input:{
age:19
}){
age
}
}
query{
account{
name
age
sex
department
}
}

添加中间件访问安全

给服务添加拦截器

使用GraphQLObjectType定义Type

  • 创建mysql数据库和表

    • npm install mysql -s

  • 这里有个坑需要修改身份认证方式否则连接不上
